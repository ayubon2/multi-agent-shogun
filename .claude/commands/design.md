# /design — 設計書閲覧・管理スキル

殿の方針・コンセプト・設計判断をまとめた設計書群を閲覧・更新する。
Memory MCP の代わりに、設計書 md ファイルを正とする。

## 手順

### Step 1: 設計書インデックスを読む

```
Read context/design/README.md
```

### Step 2: 殿に操作を聞く

`AskUserQuestion` で以下を提示:

**question**: 「設計書をどうしますか？」
**header**: 設計書
**options**:

| label | description |
|-------|-------------|
| 一覧確認 | 全設計書のインデックスを表示 |
| 読む | 特定の設計書を読んで内容を表示 |
| 更新・追加 | 既存の設計書を更新、または新しい方針を付箋として追加 |
| 全読み込み | 全設計書を一括で読み込み、現在の方針を把握（エージェント向け） |

### Step 3: 選択に応じた実行

#### 「一覧確認」の場合
README.md の内容をそのまま表示する。

#### 「読む」の場合
設計書ファイル名の一覧を表示し、殿に選ばせる。
選ばれたファイルを Read して表示。

#### 「更新・追加」の場合
1. 殿に「何について決めましたか？」と聞く
2. 既存の設計書に該当するなら Edit で更新
3. 新テーマなら新規 .md を Write で作成
4. README.md のインデックスも更新

#### 「全読み込み」の場合
`context/design/` 配下の全 .md を並行で Read し、要約を表示。
エージェントが作業前にポリシーを確認する用途。

## 注意事項

- 設計書は「付箋」。短く、1テーマ1ファイル
- 詳細な技術設計は context/vision_009.md 等に委ねる
- Memory MCP と二重管理しない。設計書が正
- 設計書の変更は git で追跡される
