# /decide — 殿への判断要請スキル

殿（ユーザー）に判断が必要な事項を選択肢形式で提示するスキル。
将軍の推奨を含めて出す。

## 手順

1. **情報収集**（すべて並行で読む）:
   - `dashboard.md` の「🚨 要対応」セクションを読む
   - `~/projects/TODO.md` から「殿の手作業」「要判断」「ブロッカー」を探す
   - `queue/reports/` 配下のレポートYAMLから `skill_candidate: found: true` を探す
   - `queue/shogun_to_karo.yaml` で status: pending の cmd で殿の判断待ちのものを探す

2. **分類と優先度付け**:
   - 🔴 ブロッカー（これがないと作業が止まる）
   - 🟡 判断待ち（殿の方針決定が必要）
   - 🟢 確認事項（進めてよいかの承認）
   - 💡 スキル候補（足軽から上がってきた再利用パターン）

3. **選択肢として提示**:
   - `AskUserQuestion` を使って最大4問まで提示
   - 各選択肢に将軍の推奨を「(推奨)」で明示
   - 判断不要なもの（既に情報が揃っていないもの等）は出さない
   - 何もなければ「現在、殿の判断を要する事項はございません」と報告

4. **判断の反映**:
   - 殿の回答を受けて、家老（karo）に `inbox_write.sh` で伝達
   - 必要に応じて `queue/shogun_to_karo.yaml` や `TODO.md` を更新
   - Memory MCP に方針として保存すべきものは保存

## 提示フォーマット例

各質問は以下の要素を含む:
- **header**: 12文字以内の短いラベル（例: "DB削除", "API選定"）
- **question**: 具体的な状況説明 + 質問
- **options**: 2-4個の選択肢。将軍推奨は最初に置き「(推奨)」を付ける
- **description**: 各選択肢のメリット・リスクを簡潔に

## 注意事項

- 技術的判断は将軍が即決する（殿に聞かない）
- 予算・方針転換・外部サービス契約・ブロッカー解消のみ殿に聞く
- 前回聞いた内容を再度聞かない（Memory MCP / dashboard の「殿が既に判断済み」を確認）
- 殿は初心者。専門用語を避け、「要するにこういうこと」を添える
