# /activity — エージェント活動モニター

全エージェントの稼働状態をリアルタイムで確認するスキル。
「動いているか？」を一目で判定する。

## 手順

### Step 1: 全エージェントのプロセス状態を取得

以下のコマンドを実行:

```bash
bash scripts/agent_activity.sh
```

### Step 2: 結果を殿に報告

スクリプトの出力をそのまま表示し、以下を添える:

1. **サマリー**: 「N名稼働中、N名待機中、N名停止」を一行で
2. **異常検知**: 以下に該当する場合は警告
   - 🔴 DEAD が1名以上 → 「{agent}が停止しています。再起動が必要です」
   - 🟡 IDLE がタスク割り当て済みなのに idle → 「{agent}にタスクが届いていない可能性あり」
   - manga-workspace の最終変更が10分以上前 → 「ファイル変更が止まっています」
3. **推奨アクション**: 異常がある場合のみ提示

### Step 3: 詳細調査（異常がある場合のみ）

🔴 DEAD の場合:
- `tmux capture-pane -t multiagent:0.{pane_index} -p | tail -5` でエラー確認
- 家老にinbox_writeで状態報告

🟡 IDLE が長時間の場合:
- `queue/tasks/{agent_id}.yaml` を確認（タスクが割り当てられているか）
- `queue/inbox/{agent_id}.yaml` を確認（未読メッセージがないか）

## 判定基準

| ステータス | CPU% | 意味 |
|-----------|------|------|
| 🟢 ACTIVE | > 5% | APIレスポンス処理中 or ツール実行中 |
| 🔵 LIGHT | 0.1-5% | API応答待ち or 軽い処理 |
| 🟡 IDLE | 0% + プロセス生存 | 入力待ち状態（タスク完了 or 未受信） |
| 🔴 DEAD | プロセスなし | Claude Codeがクラッシュまたは未起動 |

## 注意事項

- /check と異なり、こちらは「プロセスレベルの稼働確認」に特化（軽量）
- /check は通信・タスクフロー・インフラまで含む包括チェック
- CPU%は瞬間値なので、LIGHT と ACTIVE の境界は変動する
- 2回連続で実行して傾向を見るとより正確
