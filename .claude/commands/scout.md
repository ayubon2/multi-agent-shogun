# /scout — 技術スカウト網

開発品質向上・漫画サイト改善・shogunシステム高度化に使えるツール/フレームワーク/手法を探索・評価するスキル。

## 手順

### Step 1: 調査テーマ選択

`AskUserQuestion` で以下を提示:

**question**: 「何を調査しますか？」
**header**: スカウト
**options**:

| label | description |
|-------|-------------|
| Claude Agent系 | Claude Agent Teams / SDK / 公式マルチエージェント手法の最新動向 |
| 開発ツール | タスク管理・コード品質・CI/CD・テスト等の開発改善ツール |
| 漫画サイト改善 | UI/UX・画像最適化・レコメンド・SEO等のWebサイト改善ツール |
| 定期巡回 | GitHub Trending + HN + Anthropicブログを巡回して有望なものをピックアップ |

### Step 2: 忍び/軍師にタスク発行

選択に応じて家老(karo)に `inbox_write.sh` で調査指示を出す:

```bash
bash scripts/inbox_write.sh karo "<調査内容>" scout_task shogun
```

**調査の指示内容に含めるもの:**
- 調査対象（URL、キーワード、分野）
- 評価基準:
  1. shogunシステムとの適合性（tmux+YAML方式との共存/置換可能性）
  2. 導入コスト（セットアップ時間、依存関係、学習コスト）
  3. 効果見積もり（何がどれくらい改善されるか）
  4. 漫画エコシステムへの応用可能性
- 担当: 忍び(調査) → 軍師(評価) の2段階

### Step 3: 結果確認

調査結果は以下に蓄積される:
- `dashboard.md` の「🔭 技術スカウト」セクション（サマリー）
- `queue/reports/` に詳細レポートYAML
- 有望なものは殿に判断要請（T3）で提示

### 定期巡回の自動化

家老は以下のタイミングで忍びに巡回を指示:
- 空きエージェントが出たとき（他タスクより低優先）
- 夜間自律運用時の空き時間
- 殿が `/scout` で明示的に指示したとき

### 重点調査テーマ（常時）

1. **Claude Agent Teams / SDK** — 公式マルチエージェント協調フレームワーク。shogunシステムの進化先
2. **タスク駆動開発** — 例: takt (github.com/nrslib/takt) 等の構造化タスク管理
3. **漫画特化技術** — 画像CDN、カバー画像API、メタデータソース、レコメンドエンジン

### 注意事項

- 採用判断は殿確認（T3）。評価までは自律（T1-T2）
- 「使えそう」だけでなく「今のshogunの何を置き換える/補強するか」を明確に
- 流行っているだけのツールは不要。実際の課題に刺さるものだけ報告
